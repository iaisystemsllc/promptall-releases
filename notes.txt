def load_model_and_predict(texts, weights_path="model_weights.pth", meta_path="model_metadata.pkl", max_len=None):
    # Load metadata
    meta = joblib.load(meta_path)
    label_encoder = meta['label_encoder']
    word2idx      = meta['word2idx']
    max_len       = max_len or meta['max_len']

    # Build model
    device = torch.device("cuda" if torch.cuda.is_available() else "cpu")
    num_classes = len(label_encoder.classes_)
    model = BiLSTMClassifier(vocab_size=len(word2idx), embed_dim=100, hidden_dim=64, output_dim=num_classes).to(device)
    model.load_state_dict(torch.load(weights_path, map_location=device))
    model.eval()

    # Encode inputs
    def enc(s): 
        s = clean_text(s)
        tokens = [word2idx.get(w, 1) for w in s.split()]  # 1 = <UNK>
        if len(tokens) < max_len:
            tokens = tokens + [0]*(max_len - len(tokens))  # 0 = <PAD>
        else:
            tokens = tokens[:max_len]
        return tokens

    encoded = torch.tensor([enc(t) for t in texts], dtype=torch.long).to(device)

    with torch.no_grad():
        logits = model(encoded)
        probs = torch.softmax(logits, dim=1).cpu().numpy()
        preds = probs.argmax(axis=1)

    labels = label_encoder.inverse_transform(preds)
    return labels, probs

Troubleshooting quick wins

RuntimeError: expected scalar type Long...
Ensure your input ids & labels are dtype=torch.long.

index out of range in Embedding
A token id ≥ vocab_size. Rebuild vocab only from train set and keep <UNK> fallback (id=1).

CUDA out of memory
Lower batch_size (e.g., 16 or 8) or reduce max_len.

Metrics look too good / too bad
Make sure stratify=df['label'] in the split, and that test texts are not leaking into training vocab (see note above).

class BiLSTMClassifier(nn.Module):
    def __init__(self, vocab_size, embed_dim, hidden_dim, output_dim):
        super().__init__()
        self.embedding = nn.Embedding(vocab_size, embed_dim, padding_idx=0)
        self.lstm = nn.LSTM(embed_dim, hidden_dim, num_layers=1, bidirectional=True, batch_first=True)
        self.fc = nn.Linear(hidden_dim * 2, output_dim)

    def forward(self, x):
        x = self.embedding(x)
        _, (hidden, _) = self.lstm(x)
        out = torch.cat((hidden[-2], hidden[-1]), dim=1)
        return self.fc(out)

# ---------------------------
# Text pre/encoding (mirrors your script)
# ---------------------------
CLEAN_RE_1 = re.compile(r"[^A-Za-z0-9\s]")
CLEAN_RE_2 = re.compile(r"\s+")

def clean_text(text: str) -> str:
    t = (text or "").lower()
    t = CLEAN_RE_1.sub("", t)
    t = CLEAN_RE_2.sub(" ", t).strip()
    return t

def encode_sentence(sentence: str, word2idx: dict, max_len: int = 100):
    tokens = [word2idx.get(w, 1) for w in sentence.split()]   # 1 = <UNK>
    if len(tokens) < max_len:
        tokens += [0] * (max_len - len(tokens))               # 0 = <PAD>
    else:
        tokens = tokens[:max_len]
    return tokens

def load_model_and_meta():
    meta = joblib.load(MODEL_META)
    word2idx      = meta["word2idx"]
    label_encoder = meta["label_encoder"]
    max_len       = meta["max_len"]

    device = torch.device("cuda" if torch.cuda.is_available() else "cpu")
    model = BiLSTMClassifier(
        vocab_size=len(word2idx),
        embed_dim=100,
        hidden_dim=64,
        output_dim=len(label_encoder.classes_)
    ).to(device)
    model.load_state_dict(torch.load(MODEL_WEIGHTS, map_location=device))
    model.eval()
    return model, label_encoder, word2idx, max_len, device

Great question. Short answer: you can train this BiLSTM on a normal laptop. The real requirement depends on how many tickets you train on and a few model knobs. Here’s a practical sizing guide and the levers you can tune.

What actually uses memory/CPU

Embedding matrix ~ vocab_size × embed_dim × 4 bytes
e.g., 20k vocab × 100 dim × 4B ≈ 8 MB

BiLSTM weights (bidirectional, 1 layer) are small (typically <10–20 MB with embed_dim=100, hidden_dim=64).

Batch tensors (int64 ids) ~ batch_size × max_len × 8 bytes
e.g., 128 × 100 × 8B ≈ 0.1 MB per batch (tiny).

Tokenized dataset in RAM if you keep it all in memory:
num_samples × max_len × 8 bytes
e.g., 100k × 100 × 8B ≈ 80 MB (+ DataFrame overhead).

The heavy part is actually your Pandas DataFrame + text strings, not the model.

Rough hardware guidance
Small set (≤ 50k issues, avg 10–50 tokens)

RAM: 8 GB is OK, 16 GB comfy.

CPU: Any modern i5/i7/Ryzen is fine.

GPU: Optional (CPU is fine for this size).

Medium (50k–300k issues)

RAM: 16 GB recommended (32 GB nice-to-have).

CPU: Modern 4–8 cores works.

GPU: Still optional; helps if you have one. M1/M2/M3 “MPS” or a modest NVIDIA card is a bonus.

Large (≥ 300k issues or very long texts)

RAM: 32 GB recommended if you keep all tokenized data in memory.

GPU: Helpful, but not required. If CPU-only, reduce batch size/sequence length.

Disk: your model + metadata are only tens of MB. Logs/plots are tiny. Main disk use comes from any cached/serialized datasets you choose to save.

Easy knobs to fit any laptop

Limit what you pull from Elasticsearch

Start with size=50_000 (or less) and increase gradually.

Use pagination/scroll to avoid loading everything at once.

Trim sequences

max_len=100 is a good default. If logs are short, use 64; if longer, 128.

Cutting max_len has a linear impact on memory and speed.

Keep the model small

embed_dim=100, hidden_dim=64, num_layers=1, bidirectional=True → solid baseline with small footprint.

Batch size

CPU: try batch_size=64–128.

If you hit RAM limits, drop to 32 or 16.

Stream instead of hoarding

Build the vocab from a training split (one pass), then tokenize on the fly in your Dataset rather than storing all token IDs as a big NumPy array.

Minimal “stream from ES” pattern (memory-friendly)

Pass 1 (build vocab from TRAIN only): iterate ES with a generator, clean text, count words.

Pass 2 (training): re-iterate to encode each batch on the fly.

Skeleton (conceptual):

def es_docs(index, query, _source):

        yield src.get("issueDetailedDescription", ""), src.get("Label", ""), src.get("buildResID")

# PASS 1: build vocab from a capped TRAIN sample
counter = Counter()
for text, label, _ in es_docs("tickets-index", {"query": {"match_all": {}}}, ["issueDetailedDescription","Label","buildResID"]):

    text = clean_text(text)
    counter.update(text.split())

# finalize word2idx with specials
word2idx = {"<PAD>":0, "<UNK>":1}
for i, (w, _) in enumerate(counter.most_common(50000), start=2):  # cap vocab to 50k
    word2idx[w] = i
